# GIN フォーム営業オプティマイザー

企業の問い合わせフォーム経由の営業（フォーム営業）を支援するローカル完結型のビジネスツールです。
`index.html` をブラウザで開くだけで動作します（インストール・サーバー不要）。

## できること

| 機能 | 内容 |
|---|---|
| **① 設定入力** | 「どんな目的で・何を商品として・どのように売り込むのか」を入力フォーマットに沿って入力。送信者ペルソナ（会社・担当者・トーン・CTA）もここで設定 |
| **② 送り先の提案** | 目的に応じて相性の良い業種セグメントを優先度順に提案。各セグメントの読み手ペルソナ・決裁者・リストの集め方つき |
| **③ 営業文の生成** | 入力内容とセグメントに合わせて最適化されたフォーム営業文（件名＋本文500〜700字）を3トーンで生成。宛先会社を選ぶとプレースホルダを自動差し込み |
| **④ リスト管理・被り防止** | 会社リストの貼り付け登録。「株式会社/（株）/㈱」等の表記ゆれ・全半角を正規化し、**既存リスト・送信済みとの重複を自動スキップ**。送信済み/返信あり/NGのステータス管理、CSVエクスポート |
| **Claude連携** | 全設定＋宛先情報入りの最適化プロンプトをワンクリック生成（claude.aiに貼り付け）。またはAnthropic APIキーを入れてブラウザから直接生成 |

## 使い方（毎日の運用フロー）

1. **①設定入力** — 初回のみ。目的・商品・売り方・送信者情報を入力して保存
2. **②送り先の提案** — 攻める業種を決め、「リストの集め方」を参考に会社リストを収集
3. **④リスト管理** — 集めた会社を貼り付けて登録（重複は自動で弾かれる）
4. **③営業文の生成** — 宛先会社を選んで文面生成 → Claudeで最適化 → コピーして相手のフォームに貼り付け、**内容を確認して手動送信**
5. 送信したら「送信済みにする」を押す（被り防止リストに記録される）
6. 返信が来たら④でステータスを「返信あり」に。お断りされた企業は「NG」にして再送を防ぐ

## 設計方針（重要）

- **自動送信機能はありません。** 他社の問い合わせフォームへのプログラムによる大量自動送信は、フォームの利用規約違反やスパム行為に該当するリスクが高いため、このツールは「文面作成・宛先選定・記録管理」までを自動化し、送信の判断と実行は必ず人が行う設計です。
- **「営業目的の利用禁止」と明記されたフォームには送信しないでください。**
- 化粧品のため、営業文の効能表現は薬機法の範囲内に留めています（生成ルールに組み込み済み。「治る・生える・改善」はNG、「すこやかに保つ・防ぐ・うるおいを与える」まで）。
- データ（設定・リスト・送信履歴）はブラウザのローカルストレージにのみ保存されます。外部送信はClaude API利用時のAnthropic APIへのリクエストのみです。

## Claude API連携について

- 「方法B」でAPIキー（`sk-ant-...`）を入力すると、ブラウザから直接 Anthropic Messages API（モデル: `claude-opus-4-8`）を呼び出して営業文を生成します。
- キーはブラウザ内でのみ使用され、「キーを保存」にチェックした場合のみこの端末のローカルストレージに保存されます。共有PCでは保存しないでください。
- APIキーがない場合は「方法A」でプロンプトをコピーし、claude.ai に貼り付けて同等の結果が得られます。

## カスタマイズ

- 送り先セグメント（業種・ペルソナ・適合度スコア）は `index.html` 内の `SEGMENTS` 配列で定義しています。業種を追加したい場合はここに要素を足してください。
- 営業文テンプレートは `buildMessage()`、Claude用プロンプトは `buildClaudePrompt()` にあります。
